Mô phỏng lại bài toán lệnh tìm chuỗi như trong window prompt
Bài tập: viết lại lệnh find trên Windows: https://learn.microsoft.com/en-us/windows-server/administration/windows-commands/find

Các bài toàn xử lý theo các bước sau:
Mặc định : phân biệt hoa thường

1. Với câu lệnh nhập  vào dùng regex để nhận dạng câu lệnh file phân tách ra các thành phần: 0: file , 1: chuỗi tìm ,2. luật tìm"
find [/v] [/c] [/n] [/i] [/off[line]] <"string"> [[<drive>:][<path>]<filename>[...]]
 + 1	Xử lý tìm file trước với định dạng regex [<drive>:][<path>]<filename>
      Chú ý : cần xử lý bài toán ở filename có thể không rõ tên cụ thể mà là dấu * hoặc là *.txt khi đó phải lấy hết tất cả các file của folder cha ra xử lý
 + 2	Xử lý chuỗi file đi tìm  "string"
 + 3	Xử lý trình tự các case tìm với các params sau
 + 4 các kí tự [] là optional còn <> là bắt buộc , và các params có thể không theo trật tự mà có thể đổi lung tung

2. Các option params cần chú ý
 + /c /v 	hiển thị số dòng không chứa từ khóa (có phân biệt hoa thường)

		    find "hoang" /c /v C:\Users\Hoang\Downloads\test\hoang.txt
